🔧 FIRST TIME SETUP (Only Once)

If you DON'T have WSL/buildozer installed:

1. Install WSL (Windows 10/11):
   - Open PowerShell as Admin
   - Run: wsl --install
   - Restart PC
   - Open Ubuntu from Start menu

2. Install buildozer in WSL:
   sudo apt update
   sudo apt install -y python3-pip git zip unzip openjdk-17-jdk
   sudo apt install -y autoconf automake libtool pkg-config
   sudo apt install -y zlib1g-dev libncurses5-dev libncursesw5-dev
   sudo apt install -y libtinfo5 cmake libffi-dev libssl-dev
   pip3 install --user buildozer cython

3. Add to PATH (in WSL):
   echo 'export PATH=$PATH:~/.local/bin' >> ~/.bashrc
   source ~/.bashrc

📱 PHONE PREPARATION

1. Enable Developer Mode:
   Settings > About Phone > Tap "Build Number" 7 times

2. Enable USB Debugging:
   Settings > Developer Options > USB Debugging > ON

3. Enable Install from Unknown Sources:
   Settings > Security > Unknown Sources > ON
   (Or allow when prompted during install)

4. Connect to PC:
   - Use USB cable
   - Select "File Transfer" mode on phone
   - Allow USB debugging when prompted

❗ COMMON ISSUES & FIXES

Issue: "buildozer: command not found"
Fix: Install buildozer (see setup above)

Issue: "No devices found"
Fix: 
  - Check USB cable
  - Enable USB debugging on phone
  - Run: adb devices (should show your device)
  - If not, install adb: sudo apt install android-tools-adb

Issue: "Permission denied"
Fix: 
  - On phone, allow USB debugging when prompted
  - Run: adb kill-server && adb start-server

Issue: Build fails with "SDK not found"
Fix: buildozer android clean && buildozer android debug

Issue: "Java not found"
Fix: sudo apt install openjdk-17-jdk
